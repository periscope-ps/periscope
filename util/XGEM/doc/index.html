xgem

[ Top ] [ Module ]

DESCRIPTION

   The anue::xgem package provides an interface for communicating with an Anue
   XGEM box via Tcl.  This file outlines how to use this package.

USAGE

   To communicate with an XGEM box, you must follow four basic steps:
      1) Correctly import the XGEM package
      2) Connect to an Anue XGEM box
      3) Run desired commands
      4) Disconnect from the Anue XGEM box

   Each step is briefly outlined here.  A full example is given in the EXAMPLE
   section below.  The remainder of this document focuses on the details of
   each available method.
   
   1) Correctly import the XGEM package:
         Assuming you have correctly installed the Anue Tcl Library (see NOTES
         below), import the anue::xgem package using the 'package require'
         command:
            % package require anue::xgem
         If successful, the version of the Anue Tcl Library will be returned.
         
         Next, it is convenient to import the XGEM namespace:
            % namespace import ::anue::xgem::*
         This allows for easy access to the xgem commands with minimal typing.

   2) Connect to an Anue XGEM box:
         Connecting to an XGEM box returns a connection reference that is used
         to communicate with the XGEM box for the remainder of the script.
         You may create as many connection references as you want, however
         only one reference is needed for communication.
         When connecting, you must specify the connection's access privileges.
         Only one client is allowed to connect with read-write privileges.  At
         this time, connecting with read-only privileges is not supported.

         To create a connection reference and connect to an XGEM box:
            % set my_conn [open_conn -ip 192.168.40.99 -name "xgem1" -access rw]
         The name parameter (xgem1 in this example) is a friendly name used in
         various output texts to refer to the connection.

   3) Run desired commands
         Nearly all commands require a connection reference to be passed via
         the -conn command parameter:
            % set_profile_delay -conn $my_conn -line 1 -p 1 -delay 100

         If your Tcl script deals with only one connection, you may set a
         default connection used by all commands using:
            % set_default_conn -conn $my_conn
         After issuing this command, you may omit the -conn parameter from
         future commands.  Thus, the set_profile_delay command above becomes:
            % set_profile_delay -line 1 -p 1 -delay 100

         Similar commands exist for default lines and profiles: see
         set_default_profile and set_default_line.

         All commands support the -help parameter.  Passing -help will print
         the same information contained in this file for a specific command.

   4) Disconnect from the Anue XGEM box:
         To disconnect from the XGEM box before your script ends, simply call:
            % close_conn -conn $my_conn

NOTES

   The Anue Tcl Library must be installed from the Anue installation CD.
   Ensure the CD installation has completed successfully before trying to
   connect to an Anue box.

EXAMPLE

   % package require anue::xgem
   Anue Systems Inc. XGEM Version 3.2 (build 42295)
   3.2
   % namespace import ::anue::xgem::*
   % set my_conn [open_conn -ip 192.168.1.71 -name "xgem1" -access rw]
   % set_default_conn -conn $my_conn
   % set_profile_delay -line 1 -p 1 -delay 100 -unit ms
   % close_conn

set_bound_chk

[ Top ] [ xgem ] [ General ]

FUNCTION

  Set bound check error severity to a INFO or WARNING or ERROR.  The default
  severity is ERROR.
  ERRORs halt the excecution of a Tcl script.  If a command is called with an
  invalid value for a parameter (e.g. a delay that is greater than the maximum
  allowed delay), the script will stop.  If the bound check severity is set to
  WARNING or INFO, an invalid value will be automatically coerced to the
  nearest valid value and the script will continue to run.  An INFO or WARNING
  message will be displayed if a value is coerced.

SYNOPSIS

   set_bound_chk [-v|q] -severity <INFO|WARNING|ERROR>

ARGUMENTS

   -v 
     Display INFO messages
   -q
     Suppress INFO messages
   -severity
     Sets the level of error to INFO|WARNING|ERROR

RETURN VALUE

   None

OUTPUT

   [INFO] Setting bound check errors to  INFO|WARNING|ERROR

WARNINGS

   None

ERRORS

 ARG_ERR (Refer to ERROR)

EXAMPLE

   % set_bound_chk  -severity INFO ; # convert bound checks to INFO
   % ... some commands ...
   % set_bound_chk  -severity ERROR ; # revert back to bound check as ERROR

open_conn

[ Top ] [ xgem ] [ General ]

FUNCTION

   Creates a connection to a chassis and returns a connection reference.

SYNOPSIS

   open_conn [-v|q] [-nowarn_reconnect] -ip <ip_address> [-name <alias_name>]
             [-access <ro|rw>]

ARGUMENTS

   -v 
     Display INFO messages
   -q
     Suppress INFO messages
   -nowarn_reconnect
     Suppresses socket warning if socket reconnects due to inactivity. Default
     is warning enabled on reconnect.
   -ip
     The XGEM box's IP address
   -name    
     The alias name to be displayed in messages and warnings in commands. If
     not specified it is assumed to be the same as IP address.
   -access
     (ro|rw) Specifies whether the connection is read only or a read/write
     connection. A read only connection can not change settings on the device
     but can read the configuration and stats.  Defaults to rw.
     Currently "ro" connections behave as "rw" because "ro" is not implemented.

RETURN VALUE

   connection_ref 
     Reference to the connection that is created for communicating with the
     chassis

OUTPUT

   [INFO] Connected to <ip_address> (<alias_name>)  with access <access_type>
   ** The alias name is shown only if it has been specified.

WARNINGS

   None

ERRORS

 ARG_ERR, SOCK_ERR, WRONG_FW_ERR, PROTOCOL_ERR, CMD_AUTH_ERR
 TOO_MANY_CONN_ERR (Refer to ERROR)

EXAMPLE

   % package require anue::xgem
   % set my_chassis [open_conn -ip 192.168.40.99  -name "ANUE1" 
     -access rw]

set_classifier

[ Top ] [ xgem ] [ Profile Operations ]

FUNCTION

  Enables, disables and sets options for the classifier for a specific profile.
  Classifier equations are modified with set_classifier_equation.
  This command does not take effect until apply_profiles is executed.
  NOTE: Any changes to the classifier set through set_classifier or 
  set_classifier_equation are added to the current classifier configuration on
  the XGEM box.  To clear the configuration on the XGEM box, pass the -clear
  parameter to the set_classifier_equation command.

SYNOPSIS

  set_classifier [-v|q] [-conn <connection_reference>]
                 [-line <line>] [-p <profile>]
                 -enable <true|false> [-check_ip_version <true|false>]
                 [-name <profile name>]

ARGUMENTS

   -v 
     Display INFO messages
   -q
     Suppress INFO messages
  -conn
     Connection reference to send this command to
  -line 
     (Int) Line number containing the profile to use for this command, indexed
     from 1
  -p 
     (Int) Profile number to use for this command, indexed from 1
     NOTE: The default profile is profile #0, however the classifier cannot be
     configured for profile 0.
  -enable
     (true|false) Enable the particular profile
  -check_ip_version
     (true|false) When true, the IP version number is checked
  -name
     (string) A name used to describe this profile

RETURN VALUE

  None

OUTPUT

  [INFO] set_classifier on <alias_name> line <line> and profile <profile> to:
              enable           = <true|false>
              check_ip_version = <true|false>
              name             = <profile name>

WARNINGS

  None

ERRORS

  ARG_ERR, BOUND_ERR, CMD_AUTH_ERR (Refer to ERROR)

EXAMPLE

  % set_classifier -conn $my_conn -line 1 -p 1 -enable true -name "IP Traffic"
  % set_classifier_equation -conn $my_conn -line 1 -p 1 -clear
  % set_classifier_equation -conn $my_conn -line 1 -p 1
                            -protocol Ipv4
                            -field Src_Addr
                            -field_value 128...
  % set_classifier_equation -conn $my_conn -line 1 -p 1
                            -append
                            -protocol Vlan
                            -field Vlan_Id
                            -field_value 0x7F
  % set_classifier_equation -conn $my_conn -line 1 -p 1
                            -append
                            -protocol Custom_8
                            -field 0
                            -offset 24
                            -field_value 0x55
                            -field_mask 0xFF
  % apply_profiles -line 1

set_classifier_equation

[ Top ] [ xgem ] [ Profile Operations ]

FUNCTION

  Adds an element to a classifier term for a given profile.  The element
  replaces any existing elements for a term unless the -append parameter is
  passed.  If -append is passed the element is ANDed together with any existing
  elements for a term.  There is currently no support for ORing terms.
  This command does not take effect until apply_profiles is executed.
  NOTE: Any changes to the classifier set through set_classifier or 
  set_classifier_equation are added to the current classifier configuration on
  the XGEM box.  To clear the configuration on the XGEM box, pass the -clear
  parameter to the set_classifier_equation command.

SYNOPSIS

  set_classifier_equation [-v|q] [-conn <connection_reference>]
                          [-line <line>] [-p <profile>]
                          -clear
  set_classifier_equation [-v|q] [-conn <connection_reference>]
                          [-line <line>] [-p <profile>]
                          [-append]
                          -protocol <predefined protocol>
                          -field <field name>
                          -field_value <field value>
  set_classifier_equation [-v|q] [-conn <connection_reference>]
                          [-line <line>] [-p <profile>]
                          [-append]
                          -protocol <Custom_8|Custom_32>
                          -field <0-7>
                          -offset <offset>
                          -field_value <field value>
                          -field_mask <field mask>

ARGUMENTS

   -v 
     Display INFO messages
   -q
     Suppress INFO messages
  -conn
     Connection reference to send this command to
  -line 
     (Int) Line number containing the profile to use for this command, indexed
     from 1
  -p 
     (Int) Profile number to use for this command, indexed from 1
     NOTE: The default profile is profile #0, however the classifier cannot be
     configured for profile 0.
  -clear
    Clears all terms from the classifier equation for this profile
  -append
    Appends an element to existing elements for a term.  When this parameter is
    passed, the element is ANDed together with any existing elements for the
    the term.  If this parameter is omitted, this element replaces any existing
    elements in the term.
  -protocol
    (Custom_8|Custom_32|<Predefined Protocol>) Defines the protocol associated
    with this term.  See the field parameter for a list of predefined
    protocols.
  -field
    (0-7)|<Predefined Field>) Defines the field this term matches against.  If
    protocol is Custom_8 or Custom_32, this value should be a number, 0-7,
    that identifies the custom field to add.  Otherwise, acceptable predefined
    field names depend on the protocol parameter:
      Protocol    Field                      Field
      Parameter   Parameter                  Value/Type
      ------------------------------------------------------------------------
      Mac         Dest_Addr, Src_Addr            xx:xx:xx:xx:xx:xx
                  Len_Type                       Integer
      Second_Tag  Tpid, Vlan_Id, User_Priority   Integer
      First_Tag   Tpid, Vlan_Id, User_Priority   Integer
      Mpls_1      Label                          Integer
      Mpls_2      Label                          Integer
      Ipv4        Src_Addr, Dest_Addr            ***.***.***.***[/**]
                  DiffServTOS                    Integer
      Ipv6        Src_Addr, Dest_Addr            8 hexadecimal 16-bit words
                                                 seperated by colons
      Ip          Protocol                       Integer
      Tcp         Src_Port, Dest_Port            Integer
  -field_value
    When protocol is Custom_8 or Custom_32, field value is a 8-bit or 32-bit
    number to match.
    When protocol is one of the predefined protocols described above, see the
    field value/type column above for acceptable values.
    Integer values can be entered in decimal or hexadecimal.  To enter a
    number in hexadecimal, add a '0x' prefix.  String field values indicated
    above with an asterik (*) are specified in decimal, while values indicated
    above with an (x) are specified in hexadecimal (without the '0x' prefix).
    In values requiring separators (Mac address, IP address, etc.), numbers
    between separators can be left blank to not set that byte.
  -field_mask
    Only applies to Custom_8 and Custon_32 protocols.  Bit mask specifying the
    bits that should be masked when checking the value against a packet.  A
    value of '0' means that the bit is not compared and '1' means the value is
    checked.  The mask can be specified in hex using the '0x' prefix, or
    decimal.
  -offset
    Only applies to Custom_8 and Custom_32 protocols.  An offset from the
    start of the ethernet frame specified in decimal (note, there is a fixed
    offset for specifying the start of the IP layer)

RETURN VALUE

  None

OUTPUT

  [INFO] set_classifier_equation on <alias_name> line <line> and 
         profile <profile> to:
  [INFO]      append       = <true|false>
  [INFO]      protocol     = <protocol>
  [INFO]      field        = <field_name>
  [INFO]      field_value  = <field_value>
  [INFO]      field_mask   = <field_mask>
                   (** Printed if protocol = <Custom_8|Custom_32>)
  [INFO]      offset       = <offset>
                   (** Printed if protocol = <Custom_8|Custom_32>)

WARNINGS

  None

ERRORS

  ARG_ERR, BOUND_ERR, CMD_AUTH_ERR (Refer to ERROR)

EXAMPLE

  % set_classifier -conn $my_conn -line 1 -p 1 -enable true
  % set_classifier_equation -conn $my_conn -line 1 -p 1 -clear
  % set_classifier_equation -conn $my_conn -line 1 -p 1
                            -protocol Ipv4
                            -field Src_Addr
                            -field_value 128...
  % set_classifier_equation -conn $my_conn -line 1 -p 1
                            -append
                            -protocol Vlan

                            -field_value 0x7F
  % set_classifier_equation -conn $my_conn -line 1 -p 1
                            -append
                            -protocol Custom_8
                            -field 0
                            -offset 24
                            -field_value 0x55
                            -field_mask 0xFF
  % apply_profiles -line 1

set_profile_bandwidth_control

[ Top ] [ xgem ] [ Profile Operations ]

FUNCTION

  Sets the Profile Bandwidth parameters. If policer/shaper is turned off, its
  respective parameters are not required to complete the command. If the
  policer/shaper is on, then all the required parameters for that section must
  be completely specified, e.g. if policer is turned on, then burst tolerance,
  policer_rate (committed,excess,coupling) must be specified. 
  This command does not take effect until apply_profiles is executed.

SYNOPSIS

  set_profile_bandwidth_control [-v|q] [-nowarn]
                  [-conn <connection_reference>] [-line <line>] [-p <profile>]
                  -enable_policer true 
                  -policer_burst_cmtd <committed burst tolerance in bytes>
                  -policer_rate_cmtd <committed rate kbps>
                  -policer_burst_excess <excess burst tolerance in bytes>
                  -policer_rate_excess <excess rate kbps>
                  -policer_rate_coupled true|false
                  -enable_shaper true
                  -shaper_burst <burst tolerance bytes>
                  -shaper_rate <shaper rate kbps>
  set_profile_bandwidth_control [-v|q] [-nowarn]
                  [-conn <connection_reference>] [-line <line>] [-p <profile>]
                  -enable_policer true 
                  -policer_burst_cmtd <committed burst tolerance in bytes>
                  -policer_rate_cmtd <committed rate kbps>
                  -policer_burst_excess <excess burst tolerance in bytes>
                  -policer_rate_excess <excess rate kbps>
                  -policer_rate_coupled true|false
                  -enable_shaper false 
  set_profile_bandwidth_control [-v|q] [-nowarn]
                  [-conn <connection_reference>] [-line <line>] [-p <profile>]
                  -enable_policer false 
                  -enable_shaper true
                  -shaper_burst <burst tolerance bytes>
                  -shaper_rate <shaper rate kbps>
  set_profile_bandwidth_control [-v|q] [-nowarn]
                  [-conn <connection_reference>] [-line <line>] [-p <profile>]
                  -enable_policer false 
                  -enable_shaper false

ARGUMENTS

   -v 
     Display INFO messages
   -q
     Suppress INFO messages
  -nowarn
     Suppress warnings for this command (overrides defaults)
  -conn
     Connection reference to send this command to
  -line 
     (Int) Line number containing the profile to use for this command, indexed
     from 1
  -p 
     (Int) Profile number to use for this command, where the default profile is
     0 and other profiles are indexed from 1
  -enable_policer
     (true|false) Enable the profile policer
  -policer_burst_cmtd
     (Integer) Burst tolerance threshold in bytes above which packets are
     considered non compliant. Packets are considered yellow if excess policer
     rate has enough credits, else it is considered red. Red packets are 
     dropped. (** Required if enable_policer = true)
  -policer_rate_cmtd
     (Integer) Committed Policer Rate in kbps
     (** Required if enable_policer = true)
  -policer_burst_excess
     (Integer) Excess Burst tolerance threshold in bytes above which packets
     are considered non compliant. (** Required if enable_policer = true)
  -policer_rate_excess
     (Integer) Excess Policer Rate in kbps
     (** Required if enable_policer = true)
